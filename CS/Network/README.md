노드와 링크가 연결되어 있으며 리소스를 공유하는 집합.

노드 : 네트워크 장치

링크 : 유, 무선

### 좋은 네트워크란?

`많은 처리량`을 처리할 수 있고, `지연 시간` 짧음, `장애 빈도`가 적음, `좋은 보안`을 갖춘 네트워크

> 처리량
단위 시간당 전달되는 데이터 양(bps), 초당 전송, 수신되는 비트 수
트래픽, 네트워크 장치 간의 대역폭, 에러, 하드웨어 성능에 영향을 받음<br>
`대역폭` : 주어진 시간 동안 네트워크 연결을 통해 흐를 수 있는 최대 비트 수

>지연 시간
요청이 처리되는 시간
유, 무선 타입, 패킷 크기, 라우터의 패킷 처리 시간에 영향을 받음

## 네트워크 토폴로지

노드와 링크가 어떻게 배치되어 있는지에 대한 방식이자 연결 형태를 의미

### 트리 토폴로지(=계층형 토폴로지)
트리 형태로 배치한 네트워크 구성

장점 : 노드의 추가, 삭제가 쉬움.

단점 : 특정 노드에 트래픽이 집중 될 경우 하위 노드에 영향을 끼칠 수 있음

### 버스 토폴로지

중앙 통신 회선 하나에 여러 개의 노드가 연결되어 공유하는 네트워크 구성

근거리 통신망에서 사용.

장점 : 설치 비용이 적음, 신뢰성 우수, 중앙 통신 회선에 노드 추가, 삭제 용이

단점 : 스푸핑이 생길 수 있음

> 스푸핑이 뭐에요?
LAN 상에서 스위칭 기능을 마비시키거나 속여서 특정 노드에 해당 패킷이 오도록 처리하는 것.
이렇게 될 경우 일부러 악의적인 노드에 전달할 수 있음.

### 스타 토폴로지

중앙에 있는 노드에 모두 연결된 네트워크 구성

장점 : 노드 추가 및 에러 탐지 용이, 패킷의 충돌 발생 가능성 낮음, 에러 발견 쉬움, 장애 노드가 중앙 노드가 아닐 경우 다른 노드에 영향을 끼치는 것이 적음.

단점 : 설치 비용이 비쌈, 중앙 노드 장애 발생 시 전체 네트워크 사용 불가능

### 링형 토폴로지

각각의 노드가 양 옆의 두 노드와 연결하여 전체적으로 고리처럼 하나의 연속된 길을 통해 통신하는 망 구성 방식

데이터는 노드 → 노드 노드는 패킷을 처리함.

장점 : 노드 수가 증가되어도 네트워크 손실 거의 없음, 충돌 발생 가능성 적음, 노드의 고장 발견 쉬움

단점 : 네트워크 구성 변경 어려움, 회선에 장애가 발생하면 전체 네트워크에 영향을 크게 끼침.

### 메시 토폴로지(=망형 토폴로지)

그물망 처럼 연결되어 있는 구조

장점 : 하나가 장애가 발생해도 네트워크 계속 사용 가능, 트래픽 분산 처리 가능

단점 : 노드의 추가가 어려움, 구축 비용과 운용 비용이 비쌈

## 병목 현상

전체 시스템의 성능이나 용량이 하나의 구성 요소로 인해 제한을 받는 현상.

주요 원인 : `네트워크 대역폭`, `토폴로지`, `서버 CPU`, `메모리 사용량`, `비효율적인 네트워크 구성`

토폴로지가 중요한 이유는 병목 현상을 찾을 때 중요한 기준이 되기 때문임.

네트워크 토폴로지가 어떤 토폴로지를 갖는지, 어떠한 경로로 이루어져 있는지 알면 병목 현상을 올바르게 해결 가능

## 네트워크 분류

규모 : WAN > MAN > LAN (국가 및 대륙 > 도시 > 건물)

복잡도 : LAN > MAN > WAN

전송 속도 : LAN > MAN > WAN

## 성능 분석 명령어

코드상 문제가 없는데 데이터를 가져오지 못하면 네트워크 병목 현상일 가능성이 있음.

이때 네트워크 관련 테스트, 무관한 테스트를 통해 확인 후 성능 분석을 해야 함.

### ping

상태 확인하려는 대상 노드를 향해 일정 크기의 패킷을 전송하는 명령어

패킷 수신 상태, 도달하기까지의 시간, 네크워크 연결 여부를 알 수 있음.

주의점 : ICMP 프로토콜을 통해 동작함. (ICMP 프로토콜 지원 X, ICMP, traceroute를 차단하는 대상의 경우 테스트 불가능)

### netstat

접속되어 있는 서비스들의 네트워크 상태 표시

네트워크 접속, 라우팅 테이블, 네트워크 프로토콜 등 리스트를 보여줌

주로 서비스의 포트가 열려 있는지 확인할 때 사용

### nslookup

DNS에 관련한 내용을 확인하기 위한 명령어

특정 도메인에 매핑된 IP를 확인하기 위해 사용

### tracert / traceroute

윈도우 / 리눅스

목적지 노드까지 네트워크 경로를 확인할 때 사용하는 명령어

목적지 노드까지 구간들 중 어느 구간에서 응답 시간이 느려지는지 등을 확인할 수 있음.

## 프로토콜 표준화

>네트워크 프로토콜이 뭐에요?
다른 장치들끼리 데이터를 주고받기 위해 설정된 공통된 인터페이스
IEEE or IETF라는 표준화 단체가 정함.<br>
IEEE802.3 : 유선 LAN 프로토콜, 유선으로 LAN을 구축할 때 쓰이는 프로토콜
HTTP : 웹 서비스를 기반으로 데이터를 주고 받을 수 있음.
